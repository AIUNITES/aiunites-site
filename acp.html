ACP — AI Context Protocol v0.1

You Are a Different Person
in Every AI Tool.

Claude doesn't know what you know. ChatGPT doesn't know what Claude learned. Every session starts from zero. ACP is an open protocol for portable AI context — your preferences, projects, persona, and terminology carried across every AI tool you use.

Read the Spec → Build Your ACP

🛑 The Problem

Every AI tool you open is a blank slate. It doesn't know you work in higher education. It doesn't know you prefer concise answers. It doesn't know your company is called AIUNITES, that you have 22 sites, or that you write in plain HTML with no frameworks. You re-explain yourself hundreds of times a year.

This isn't a technical limitation — it's a missing layer. There is no standard for what a user is to an AI tool. No portable format. No shared protocol. The AI tools are all built, but the middleware between you and all of them doesn't exist yet.

✓ The ACP Solution

ACP is a structured JSON document that describes you to any AI. You build it once. You paste it (or inject it automatically via MCP) into any AI session. The AI immediately knows your context, domain, preferences, ongoing projects, and communication style.

Open format • No vendor • Works with Claude, GPT-4, Gemini, and any LLM that accepts a system prompt.

The ACP Specification

Version 0.1 — Draft. Open for community input.

// ACP — AI Context Protocol v0.1 // Paste this block as a system prompt or user message to any AI tool. // Full spec: https://aiunites.com/acp.html { "acp": "0.1", // WHO YOU ARE "identity": { "name": "Tom", "role": "Founder, AIUNITES LLC", "domain": "AI middleware, web publishing, SIS database consulting", "location": "Tulsa, Oklahoma", "expertise_level": "senior" // junior | mid | senior | expert }, // HOW YOU COMMUNICATE "preferences": { "response_length": "concise", // brief | concise | detailed | comprehensive "tone": "direct", // formal | professional | direct | casual "code_style": "vanilla JS, no frameworks, dark themes, CSS variables", "avoid": ["unnecessary preamble", "excessive caveats", "asking permission to proceed"], "prefer": ["direct answers", "file edits over suggestions", "ASCII-only in scripts"] }, // YOUR CURRENT PROJECTS "projects": [ { "name": "AIUNITES Network", "description": "22-site GitHub Pages network with autonomous publishing pipeline", "stack": ["HTML", "Vanilla JS", "PowerShell", "GitHub Pages"], "status": "active" } ], // YOUR TERMINOLOGY (so the AI uses your words, not generic ones) "terminology": { "Publish Now": "click Run in System Monitor tray to trigger auto-publish.ps1", "BODWAVE": "our trademark for the Mind-Neuro-Muscle training methodology", "webring": "the fixed top nav bar linking all 22 AIUNITES sites together" }, // CONSTRAINTS THE AI SHOULD ALWAYS RESPECT "constraints": { "never_use": ["display:none on nav elements", "em dashes in PowerShell", "npm or framework dependencies"], "always_use": ["MCP filesystem tools for file writes", "ASCII-only in .ps1 scripts"] }, // OPTIONAL: WHAT THE AI SHOULD KNOW ABOUT THIS SESSION "session": { "focus": "Building the ACP spec and profile builder", "context": "Continuing from previous session — pipeline scripts deployed, pricing on consulting page" } }

Field Reference

FieldTypeDescriptionRequired?
acpstringProtocol version. Always "0.1" for this spec.Required
identity.namestringYour name or handle. Used in AI salutations and references.Required
identity.rolestringYour job title or primary function. Sets AI's mental model of your work context.Required
identity.domainstringYour field(s) of work. The AI uses this to calibrate assumed knowledge level.Required
identity.locationstringCity/region. Used for timezone, locale, and market-specific advice.Optional
identity.expertise_levelenumjunior / mid / senior / expert. Calibrates explanation depth.Optional
preferences.response_lengthenumbrief / concise / detailed / comprehensiveOptional
preferences.toneenumformal / professional / direct / casualOptional
preferences.code_stylestringFree text describing your coding conventions and preferences.Optional
preferences.avoidarrayBehaviours the AI should suppress in every response.Optional
preferences.preferarrayBehaviours the AI should prioritise.Optional
projectsarrayActive projects with name, description, stack, and status. The AI uses this to understand what you're building.Optional
terminologyobjectKey/value pairs of your domain-specific terms. The AI uses your vocabulary, not generic synonyms.Optional
constraints.never_usearrayHard rules the AI must always respect (tech choices, patterns to avoid).Optional
constraints.always_usearrayHard rules the AI must always follow (tools, patterns, conventions).Optional
sessionobjectSession-specific context: what you're working on right now, what happened last time. Rotate this per session.Optional

How to Use Your ACP

📋

Paste as System Prompt

Copy your ACP JSON and paste it at the start of any AI conversation. Works with Claude, ChatGPT, Gemini, and any tool that accepts a system or first message.

"Here is my ACP context: [paste JSON]"
🔗

Claude MCP Server

AIUNITES is building an MCP server that auto-injects your ACP into every Claude session. Your context loads before you type your first message.

mcpServers: { "acp": { "url": "..." } }
📄

Save as a File

Store your ACP as my-acp.json and upload it to any AI that accepts file attachments. Update it as your projects evolve.

C:\Users\you\Documents\my-acp.json
⚙️

Custom Instructions

Paste a condensed version of your ACP into Claude's "Custom Instructions" or ChatGPT's "Memory" settings for always-on context.

Settings → Custom Instructions → paste ACP
